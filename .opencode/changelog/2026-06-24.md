# Changes - 2026-06-24

## Batch Rejection Summary Emails

- **Why**: Bulk rejecting submissions sent one rejection email per submission and hit Resend's 5 requests/second limit.
- **What**: Bulk reject updates now tag records with a batch id, PocketBase queues rejected records by creator, and a flush route sends one summary email per creator with a small delay between sends.
- **Files**: `web/backend/pb_hooks/submission_update_email.pb.js`, `web/src/hooks/use-submissions.ts`
- **Tests**: `pnpm --dir web test` passed; focused Biome and hook syntax checks passed. Full lint/typecheck/build still fail on pre-existing unrelated issues.
- **Docs**: Changelog added; no AGENTS.md present or needed.
- **Risks**: The batch queue is in-memory, so a PocketBase restart between updates and flush would drop the grouped email.
